# Contributing to Wikipedia LLM Training

Thank you for your interest in contributing to the **Wikipedia LLM Training** repository! We welcome contributions from the community to improve the training pipeline, add features, fix bugs, or enhance documentation.

## How to Contribute

### 1. Reporting Issues
- Check the [issue tracker](https://github.com/VoxDroid/llm-wikipedia/issues) to ensure the issue hasn’t been reported.
- Use the appropriate issue template (bug report, feature request, or documentation improvement) in `.github/ISSUE_TEMPLATE/`.
- Provide clear details, including steps to reproduce, expected behavior, and actual behavior.

### 2. Submitting Pull Requests
- Fork the repository and create a new branch for your changes (`git checkout -b feature/your-feature-name`).
- Follow the existing code structure and style.
- Test your changes to ensure they work correctly.
- Submit a pull request using the [PULL_REQUEST_TEMPLATE.md](https://github.com/VoxDroid/llm-wikipedia/blob/main/.github/PULL_REQUEST_TEMPLATE.md).
- Ensure your code adheres to Python best practices and includes necessary documentation.

### 3. Code Style
- Follow PEP 8 for Python code.
- Use clear, descriptive variable names and comments.
- Ensure compatibility with Python 3.8+.
- Include dependencies in `requirements.txt` or similar.

### 4. Testing
- Test your changes on a clean Python environment.
- Include any setup instructions in the documentation.
- If applicable, add unit tests.

### 5. Documentation
- Update relevant documentation files with clear explanations.
- Suggest improvements to the main `README.md` or other docs if needed.

## Getting Help
If you need assistance, refer to [SUPPORT.md](SUPPORT.md) or open an issue with the “question” label.

## Code of Conduct
All contributors must adhere to our [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

Thank you for helping make **Wikipedia LLM Training** better!
